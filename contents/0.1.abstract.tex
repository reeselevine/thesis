% Abstract

Graphics Processing Units, or GPUs, have emerged as powerful parallel computing platforms, offering significant performance advantages for a wide range of applications beyond traditional graphics rendering. To harness this power, GPU frameworks provide a parallel programming model that defines the semantics of execution. One aspect of a GPU programming model is its memory consistency specification (MCS), which describes the behavior of concurrent memory operations. The MCS provides rules necessary for predictable execution on parallel hardware, and also interacts with other aspects of the programming model to describe lanaguage-level security guarantees like memory safety, i.e., preventing access to out-of-bounds or uninitialized memory. Validating both that implementations conform to the programming model's MCS and that the MCS accurately describes the behavior of real machines is crucial for ensuring correctness and safety of GPU applications.

This thesis develops several techniques and large-scale studies for validating GPU memory consistency and memory safety. First, it introduces \textit{MC Mutants}, a mutation testing framework for GPU memory consistency models that systematically generates mutants of small concurrent GPU programs and evaluates whether testing environments detect them. MC Mutants enables new empirical approaches for validating GPU memory consistency models and uncovers several bugs in GPU programming models and their implementations. Next, \textit{GPUHarbor} is presented, a tool for large-scale testing of GPU memory consistency. GPUHarbor facilitates the largest study of GPU memory consistency to date across 106 devices, uncovering multiple memory consistency bugs and providing insights into the behavior of diverse GPUs. The techniques that allow GPUHarbor to scale to numerous GPUs are also applied to other GPU frameworks and vendors, analyzing precisely the architectural components of several NVIDIA GPUs and helping to discover a new class of hardware sidechannels across vendors based on observations of certain memory consistency behaviors. Finally, \textit{SafeRace} is described, a collection of security assessments and specification proposals aimed at improving memory safety in the WebGPU programming model. SafeRace assessments identify memory safety vulnerabilities in GPUs from several vendors, and its proposals provide a validated path toward stronger memory safety guarantees in WebGPU.
