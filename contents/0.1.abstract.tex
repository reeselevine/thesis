% Abstract

Graphics Processing Units, or GPUs, have emerged as powerful parallel computing platforms, offering significant performance advantages for a wide range of applications beyond traditional graphics rendering. To harness this power, compute-focused GPU frameworks have been developed, providing a general-purpose parallel programming model. One aspect of these programming models are their memory consistency specifications (MCS), which describes the behavior of concurrent memory operations. The MCS defines the semantics of concurrent shared-memory operations, and also interacts with other aspects of the programming model to describe language-level security guarantees like memory safety, i.e., preventing access to out-of-bounds or uninitialized memory. Validating both that implementations conform to an MCS and that the MCS accurately describes the behavior of real machines is crucial for ensuring correctness and safety of GPU applications.

This thesis develops several techniques and large-scale studies for validating GPU memory consistency and memory safety. First, it introduces \textit{MC Mutants}, a mutation testing methodology for GPU MCSs that systematically generates mutants of small concurrent GPU programs and evaluates whether testing environments detect them. MC Mutants enables new empirical approaches for validating GPU MCSs and uncovers several bugs in the WebGPU MCS and its implementations. Next, \textit{GPUHarbor} is presented, a tool for large-scale testing of GPU MCSs. GPUHarbor facilitates running tests across 106 devices, uncovering multiple memory consistency bugs and providing insights into the behavior of diverse GPUs. The lessons learned while developing the tooling for GPUHarbor also transfer to several other studies, including an analysis of the architectural components of several NVIDIA GPUs and the discovery of a new class of hardware side-channels across vendors based on observations of certain memory consistency behaviors. Finally, \textit{SafeRace} is described, a collection of security assessments and specification proposals aimed at improving memory safety in WebGPU. SafeRace assessments are run across dozens of GPUs and identify memory safety vulnerabilities in GPUs from several vendors, and its proposals provide a validated path toward stronger memory safety guarantees in WebGPU.
